Go语言之路第二章程序结构

发表于 2017-12-09 | 分类于 读书笔记 |

一月之后就要开始自己的Go语言之路了，2018将会是一个新的开始．在接下一个月里需要好好的准备一下Go语言，争取到下家之后可以直接上收干活儿，而不需要花费太多的时间在语言上面．这里前期主要参考的是Donovan和Kernighan的大作The Go Programming Language,　后期有时间的话再找一个小型的开源项目来学习一下语言的具体实战．

从设计者的角度来看的的话Go是完全集成了C的优点，想较于面向对象中一切都是对象，基本对全局的变量持否定态度，在Go中并没有这样的规定，在Go中如果一个Entity声明是在一个函数里面的话，这个变量函数局部是可见的，在函数外是不可见的，如果声明函数外面的话是整个package可见的，再者就是Go中并没有专门的export方法，如果一个函数第一个字母是以大写开始的那么这个方法就是export出去的，其他程序中import相应的包之后就可以使用了．我们可以拿fmt包来看看，其中export的函数是:

Go变量的声明

Go中声明变量的一个变量的格式如下：
　　　var name type = expression
其中type和= expression两者可以选择一个，如果只有type就用对应类型的零值来代替，如果只有= expression那么就根据值来推到出他的类型.

在Golang中使用如下的零值规则:

  • string的零值为””
  • 数值类的零值为0
  • 其他类(slice, pointer, map, channel, function)的零值为nil
    这里的零值规则也就保证里变量肯定是被初始化的，不会出现其他语言里的变量为初始化异常．

变量还有另外一个更简单的声明方式就是 name := expression,尤其是对一个local的变量是尤为推荐的，而且这里的name一定要简单，i, j啥的应该是最好了.其实在之前第一章中我们就看到了简式声明在for循环中的应用:

Go中的指针

当再一次在高级语言中看到指针这两个字的时候多少还是有点惊讶的．看完这一小节之后你就会发现Go真的是完全集成了C中的指针概念．想当初指针，地址折磨了多少学生啊.
指针其实让我们回到了一个最原始的问题上，计算机是怎么存储数据的以及这些数据是如何管理的．这里的可以参考pointer in C.

Go中的pacakge，import以及其他

Go的package类似于其他语言中的module或者lib，一个package由多个文件组成，然后地一个大写字母的函数是export出去的，比如fmt包就有如下的几个文件组成:
doc.go
export_test.go
fmt_test.go
format.go
print.go
scan.go
scan_test.go
stringer_test.go

除了语言自带的pacakge之外，我们import的时候还需要带上他的相对路径比如　“gopl.io/ch2/tempconv”其中tempconv就是包的名字.
包的初始化可以使用init()函数来完成:

One package is initialize d at a time, in the order of imports in the program, dependencies firs t, so a package p importing q can be sure that q is fully initialized before p’s initialization begins. Initialization proceeds from the bottom up; the main package is the last to be initialized. In this manner, all packages are fully initialized before the application’s main function begins.
